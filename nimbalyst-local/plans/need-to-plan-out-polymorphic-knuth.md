## Approach

Four steps, in order. Scripts already exist — this is mostly running, verifying, and lightly enriching.

### Step 1 — Verify Salesforce auth + automation are live
- Confirm `~/.claude/sf_tokens.json` exists (Salesforce authenticated once). If missing, run `sf_authenticate` via the Salesforce MCP first — every downstream script depends on it (`.scripts/automation/README.md:55`).
- Run `pwsh .scripts/automation/register-automation.ps1 -Status` to confirm the three tasks (`Dex-Weekly-SF-Sync` Mon 06:00, `Dex-Weekly-Lease-Alert` Mon 06:30, `Dex-Monthly-Intel-Report` 1st 08:00) are registered with next-run times.
- If not registered, run `register-automation.ps1` (idempotent) and re-check `-Status`.

### Step 2 — Refresh data to today, then re-score
- Run `. .scripts/automation/run-sf-sync.ps1` to pull a fresh local Salesforce cache now (don't wait for Monday). Confirm `.scripts/salesforce-data/manifest.json` timestamp updates.
- Re-run `python .scripts/customer-intel/score-key-accounts.py` to regenerate `Key_Accounts_H2-2026.md` dated **today (July 1)**. Diff the Top-5 against the June-30 version to confirm the pipeline still looks right.

### Step 3 — Enrich top Tier-1 equipment intel (coverage, not weighting)
The Equipment/Expiry column is blank for most Tier-1 accounts because the EDA join is **plasma-only** (`score-key-accounts.py:162–189`). This is a *coverage* gap, not a scoring one — so it doesn't touch the weights.
- For the top ~10 Tier-1 accounts, pull live equipment via `mcp__salesforce__sf_get_account_assets` (all machine types, incl. competitor) and confirm/annotate the "Equipment / Expiry" and "Why now" context.
- Apply the **Date Accuracy Protocol** to every expiry date before it informs outreach.
- Keep this as an annotation layer on the ranked list (and the relevant account/project pages) — do **not** re-weight scores.

### Step 4 — Stamp activation status + close the loop
- Add a short "Activation Status" note (last-verified date, task state, last data refresh) to `Planning/2026-H2_Sales_Plan.md` and/or `.scripts/automation/README.md` so it's obvious the system is live.
- Confirm the weekly operating loop is ready to run (`/week-plan → /week-itinerary → outreach wave → /process-meetings → /pipeline-review`). No new automation for the send/log steps — those stay human-approved by design.

**Explicitly out of scope (per decisions):** changing scoring weights; auto-ranking by deal size; any job that auto-sends email or writes to Salesforce.
