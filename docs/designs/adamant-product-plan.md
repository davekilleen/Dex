## Vision

### 10x Check
Adamant as the product development nervous system — not just a feedback widget, but a unified intent layer that captures signals from prototype submissions, support tickets, sales calls, usage analytics, and internal discussions. The prototype widget is one channel, not the whole product. No one else unifies all sources of product intent into a single structured layer.

### Platonic Ideal
A product leader opens Adamant and sees a live map of their product. Every surface glows with signal density. They click on the checkout flow — bright red. They see: 47 prototype submissions, 12 support tickets about confusion at step 3, usage data showing 34% drop-off, and 3 Slack threads from engineers. All clustered automatically. All classified. They click "Generate Roadmap Evidence Pack" and get a document proving what users want with evidence. The feeling: "I finally understand what my users want, and I can prove it."

## Key Architectural Decisions

### Lean v0 Architecture
- **Stack:** Next.js + Supabase (Postgres) + R2 (storage) + Claude API (LLM)
- **Deploy:** Vercel + Supabase. ~$50-200/month at pilot scale.
- **No enterprise infra for v0:** No Kafka, no Firecracker, no Neo4j, no EKS.
- **Migration triggers:** >1000 signals/day → queue, >10 tenants → isolation, PHI/SOC2 → server sandbox, >50 tenants → EKS.

### Prototype Sandbox: Hybrid
- Server generates + validates the prototype (LLM → IR → validation).
- Client renders interactively in sandboxed iframe.

### Design System: Code-first, Figma Advisory
- Code components (Storybook/React) are the rendering source of truth.
- Figma used for discovery and naming alignment only.
- Drift warnings shown but prototypes always render from code.

### Widget Distribution: Dual Channel
- Browser extension for MedWrite internal pilot (fastest, no code changes).
- JS SDK (script tag) for external design partners (like Intercom).

### Product Identity: Tool Positioning, Infrastructure Architecture
- Sell as a PM tool (feedback capture + review). Easy pitch, fast adoption.
- Build as infrastructure (data model, APIs, extensibility) to reposition later.

### PHI/HIPAA: Active Detection
- PHI detection layer scans feedback text before storage.
- Mock-only data in prototypes (no real patient data).
- BAA-ready storage (encrypted, access-logged).

### Error Handling: Never Lose Feedback
- Fallback chain: retry → wireframe fallback → text-only capture.
- User's input is always preserved even if prototype generation fails.
